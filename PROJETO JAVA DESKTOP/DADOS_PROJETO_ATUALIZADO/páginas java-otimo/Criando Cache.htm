Criando um Cache

em 27/08/2002 , por Paulo Silveira
O LinkedHashMap
Este é um simples tutorial para que você possa criar um sistema de Cache eficiente e muito simples. O Cache que queremos fazer é um que siga o modelo LRU (least-recently-used), isto é, ele guarda N itens, e na hora que esse N fica maior que o MAX que você deseja ter em memória, o item que foi menos acessado é removido. Este tipo de Cache é muito interessante para resultados de SELECTs que são muito frequentemente chamados, ou na hora de gerar um XML por exemplo. Você pode utilizar este tipo de Cache também para guardar os 10 últimos arquivos mais utilizados. As utilização são milhares. Para a construção deste Cache, vamos utilizar um novo Map que foi inserido no java.util no jdk1.4: o LinkedHashMap. A grande diferença entre ele e um HashMap comum, é que se você pegar o Iterator do LinkedHashMap.keySet(), você irá iterar sobre os elementos em uma determinada ordem, e não em uma ordem imprevisível, como acontece no HashMap. Esta ordem pode ser duas: ordenado por inserção de elementos, ou ordenado pela utilziação de elementos. Repare a utilização do LinkedHashMap ordenado por inserção (default).
view plainprint?
  1. Map map = new LinkedHashMap();  
  2. map.put("1", null);  
  3. map.put("2", null);  
  4. map.put("3", null);  
  5. map.put("4", null);  
  6. map.put("5", null);  
  7.   
  8. Iterator i = map.keySet().iterator();  
  9. while (i.hasNext()) {  
  10.     System.out.println(i.next());  
  11. }  

A saída virá na ordem da inserção!
Criando o cache
Além desta diferença, o LinkedHashMap possui um método removeEldestEntry, que simplesmente retorna false. Se ele for sobrecarregado, e retornar true em algum momento, um elemento será jogado fora do Map. O elemento a ser jogado fora depende da ordenação do LinkedHashMap, definida em sua construção. Com isso, conseguimos criar o Cache que desejavamos:
view plainprint?
  1. public class Cache extends java.util.LinkedHashMap {  
  2.     // poderia ser final:  
  3.     protected int maxsize;  
  4.   
  5.     public Cache(int maxsize) {  
  6.         super(maxsize, 0.75f, true);  
  7.         this.maxsize = maxsize;  
  8.     }  
  9.   
  10.     /** 
  11.      * Faz com que o elemento mais velho 
  12.      * seja removido no caso do tamanho ser atingido 
  13.      */  
  14.     protected boolean removeEldestEntry() {  
  15.         return (this.size() > maxsize);  
  16.     }  
  17. }  

Realmente não teria como ser mais simples, não? O super() deve ser chamado com os 3 parâmetros, pois o 3o argumento indica que a lista deve ser mantida na ordem de acesso aos elementos. Se você passar false, ela vai ser mantida na ordem de inserção, e na hora de remover um elemento, ele irá remover o último inserido, algo que não faz sentido na política de um cache.
